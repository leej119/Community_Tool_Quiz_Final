# Care Load Calculator

## Project Overview
Care Load Calculator is an interactive self-assessment website that helps users reflect on how much emotional labor they may be carrying in tense or polarized everyday interactions. The tool is designed for students, friends, family members, and collaborators who often find themselves softening their language, staying quiet to avoid escalation, or trying to keep conversations stable.

Rather than asking users about political beliefs directly, the website focuses on behavior: how people respond to tension, disagreement, and discomfort. Users answer 8 questions, and based on their responses the site generates a score and a result page with practical suggestions and conversation scripts.

This project was created for Project 4: Community Tool or Quiz using HTML, CSS, and JavaScript.

---

## Community / Audience
This project is for people who often take on the emotional work of maintaining stability in:
- group projects
- family conversations
- friendships
- shared living situations
- class discussions

I chose this group because it connects directly to my own experience and interests. I am interested in how instability, negotiation, and uneven responsibility appear in shared environments. This project translates those concerns into a practical tool that users can engage with in real life.

---

## Project Purpose
The purpose of this tool is to make invisible emotional labor more visible. In difficult interactions, one person often adjusts more than others by mediating tension, preventing escalation, or carrying the discomfort of the situation. This labor is often not recognized, even though it affects how people communicate and how they feel afterward.

This website gives users a way to notice these patterns in themselves. It does not diagnose or solve polarization, but it offers a small intervention by helping users identify whether they may be carrying a high, moderate, or lower care load and by giving them language they can use in future conversations.

## Design Process
My first idea was more abstract and connected to my studio practice around instability, negotiation, and uneven distribution of responsibility. I was interested in how some people absorb more tension than others in shared systems.

As the project developed, I shifted the idea into something more practical and usable in everyday life. Instead of making a purely reflective artwork, I turned it into a self-assessment tool. I focused on situations people can recognize immediately, such as:
- trying to keep conversations calm
- adjusting tone to avoid conflict
- holding back an opinion
- feeling drained after difficult interactions

I wanted the design to stay visually simple but still connected to my conceptual interests. The circle-based answer buttons were inspired by a clover-like arrangement instead of standard rectangular quiz buttons. This made the interface feel softer and more relational while still remaining functional.

The final website has four sections:
1. Introduction
2. Quiz
3. Project purpose and research
4. Result page

## Inspiration
This project was inspired by Caroline Sinders’ idea of **useful art**. In *In Defense of Useful Art*, Sinders argues that art does not need to be separate from function, and that artistic work can take the form of tools, systems, workshops, or open-source interventions that people can use in real life.

That idea helped shape this project. I wanted the website to function not only as a representation of tension or emotional labor, but as a small practical tool that users can interact with directly.

The project is also informed by my broader artistic interest in:
- instability
- relational negotiation
- uneven responsibility
- how shared systems are maintained through subtle bodily and emotional adjustments

## How the Website Works
The website was built using only HTML, CSS, and JavaScript.

### Main features
- 8 self-assessment questions
- radio-button style input using circle-shaped choices
- localStorage to save user answers
- progress bar that updates as the user answers questions
- score calculation based on user responses
- result page that changes content and page background based on score
- practical scripts users can apply in real conversations

### Scoring system
Each answer has a score:
- Often = 2
- Sometimes = 1
- Rarely = 0

The total score determines the user’s result:
- **11–16** = High care load
- **6–10** = Moderate care load
- **0–5** = Lower reported care load

## Challenges
One challenge was balancing concept and usability. My initial ideas were more abstract and closer to my studio practice, but the assignment required a tool that could function for a specific community in a practical way. I had to rethink how to translate a conceptual interest into a usable quiz.

Another challenge was writing questions that were specific enough to feel real but still broad enough to apply across different situations like family, class, friendship, or group work.

A design challenge was creating a layout that felt visually connected to my interests without making the tool confusing to use. I wanted to avoid a generic quiz format, so I experimented with circular answer choices and a clover-like structure while still keeping the interface clear.

A technical challenge was managing localStorage and making sure saved answers updated the progress bar, score, and result page correctly.

## User Testing
For user testing, I would ask a few classmates or friends to go through the site and respond to these questions:
- Were the quiz questions easy to understand?
- Did the result feel accurate or relatable?
- Was the interface clear and easy to navigate?
- Did the project feel useful in a real-life situation?
- What would make the tool more helpful?

Their feedback would help me revise unclear language, improve the pacing of the site, and refine the result descriptions.
